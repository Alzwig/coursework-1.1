Назначение скрипта:
Скрипт автоматизирует создание коммитов для исходного кода и формирование патча, содержащего все изменения в проекте с первого коммита до текущего состояния. Патч всегда сохраняется рядом со скриптом, независимо от того, из какой директории он был запущен.

Как работает скрипт:
-Определяет папку, в которой находится сам скрипт.
-Определяет путь к каталогу с исходным кодом относительно скрипта.
-Проверяет, есть ли изменения в исходниках.
-Если изменения есть, создаёт новый коммит формата comm:N, где N автоматически увеличивается.
-Создаёт патч src.patch — разницу между первым и последним коммитом.
-Сохраняет патч рядом со скриптом, вне зависимости от текущей рабочей директории.

Структура проекта:

coursework/
 ├── make_patch.sh
 └── Surname_Name_cw/
     └── src/
         └── (файлы с кодом)

Как пользоваться:
1. Поместить make_patch.sh в папку, где будет создан репозиторий.
2. В скрипте изменить путь до папки с кодом относительно скрипта (это строка 6, отмеченная комментарием).
3. Создать локальный репозиторий или клонировать существующий:
git init
# или
git clone <url>
4. Создать путь к каталогу с исходниками (например, по требованию курсовой Surname_Name_cw/src).
5. Создать пустой файл main.c в папке src.
6. Сделать первый коммит:
    git add main.c
    git commit -m "Init"
7. Теперь можно пользоваться скриптом:
    - перейти в папку, где лежит скрипт (в примере — coursework);
    - выдать права на выполнение:
      chmod +x ./make_patch.sh
    - запустить скрипт:
      ./make_patch.sh

Примечение:
Скрипт можно запускать из любой директории — он корректно работает независимо от текущего пути.
Файл патча всегда будет создан рядом с файлом make_patch.sh
